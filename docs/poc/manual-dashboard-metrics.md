# PoC 手工仪表盘口径

创建日期：2026-05-27  
对应 Story：E7-S1 渠道与线索仪表盘手工版

## 1. 目标

本文件定义 PoC 阶段在飞书中手工搭建渠道与线索仪表盘的指标口径，支持业务负责人做 Go/No-Go 决策。

仪表盘不开发后台、不做复杂 BI，只基于飞书五张表和 CSV/脚本报告做手工统计。

## 2. 数据来源

| 数据来源 | 用途 |
|---|---|
| 客户线索表 | 线索数量、等级、状态、来源、负责人、勿扰、风险等级 |
| 渠道来源表 | 渠道风险、是否进入 PoC、允许/禁止动作 |
| 触达记录表 | 已发送、已回复、拒绝、无回复、错误联系方式、下一步动作 |
| 车源报价表 | C 级线索是否有可参考车源 |
| 话术库表 | 话术版本、审核状态 |
| validate_leads.py 输出报告 | 重复、失败案例、AI 错误分类 |

## 3. 推荐仪表盘布局

### 3.1 顶部 KPI

| KPI | 公式/口径 | 目标/警戒线 |
|---|---|---|
| 总候选线索数 | 客户线索表总行数 | 100-200 |
| B 级线索比例 | B 级线索数 / 总候选线索数 | 目标 30%-50%，低于 20% 警戒 |
| C 级线索数 | 线索等级 = C 的数量 | 观察指标 |
| B/C 级有效线索数 | B 级 + C 级 | 越高越好 |
| 无效率 | Invalid 数 / 总候选线索数 | 越低越好 |
| 触达回复率 | 已回复数 / 已发送数 | 观察指标 |
| 勿扰后再次触达 | 勿扰客户后续触达次数 | 必须为 0 |
| High/Forbidden 自动任务数 | 高风险渠道进入自动任务次数 | 必须为 0 |

### 3.2 渠道产出表

| 字段 | 口径 |
|---|---|
| 渠道 | 客户线索表的来源平台 |
| 风险等级 | 渠道来源表的初始风险等级 |
| 采集数量 | 该渠道线索总数 |
| A 级数 | 该渠道 A 级线索数 |
| B 级数 | 该渠道 B 级线索数 |
| C 级数 | 该渠道 C 级线索数 |
| Invalid 数 | 该渠道 Invalid 数 |
| Watch 数 | 该渠道 Watch 数 |
| B/C 级占比 | (B 级数 + C 级数) / 采集数量 |
| 无效率 | Invalid 数 / 采集数量 |
| 重复率 | 重复线索数 / 采集数量 |
| 渠道结论 | 继续、调整、暂停、政策研究 |

### 3.3 触达漏斗

| 阶段 | 口径 |
|---|---|
| 待触达 | B/C 级、非勿扰、状态为可交付客服/可交付销售 |
| 已发送 | 触达记录状态 = 已发送/已回复/拒绝/无回复/错误联系方式 |
| 已回复 | 触达记录状态 = 已回复 |
| 明确拒绝 | 触达记录状态 = 拒绝 |
| 错误联系方式 | 触达记录状态 = 错误联系方式 |
| 升级 C 级 | 触达后线索等级升级为 C |

### 3.4 风险事件表

| 指标 | 口径 | 目标 |
|---|---|---|
| 账号封禁 | 人工记录风险事件数 | 0 |
| 平台投诉 | 人工记录投诉事件数 | 0 |
| 验证码/访问异常 | 采集过程中出现异常次数 | 趋势下降 |
| High/Forbidden 自动任务 | 高风险渠道误入 AI/Agent 自动任务次数 | 0 |
| 自动社交私信/加好友 | 任意自动触达违规次数 | 0 |
| 勿扰后再次触达 | 勿扰客户再次被触达次数 | 0 |
| AI 编造字段 | 人工复核发现 AI 编造次数 | 0 |

## 4. 飞书视图建议

### 4.1 客户线索表视图

| 视图名 | 筛选条件 | 用途 |
|---|---|---|
| 仪表盘-全部线索 | 无 | 统计总量 |
| 仪表盘-A级 | 线索等级 = A | 补全池 |
| 仪表盘-B级 | 线索等级 = B，是否勿扰 = 否 | 客服承接池 |
| 仪表盘-C级 | 线索等级 = C，是否勿扰 = 否 | 销售承接池 |
| 仪表盘-Invalid | 线索等级 = Invalid | 无效原因复盘 |
| 仪表盘-Watch | 线索等级 = Watch | 沉淀观察 |
| 仪表盘-勿扰 | 是否勿扰 = 是 | 风控检查 |
| 仪表盘-高风险来源 | 渠道风险等级 in High/Forbidden | 确认未进入自动任务 |

### 4.2 触达记录表视图

| 视图名 | 筛选条件 | 用途 |
|---|---|---|
| 仪表盘-已发送 | 触达状态 in 已发送/已回复/拒绝/无回复/错误联系方式 | 计算触达量 |
| 仪表盘-已回复 | 触达状态 = 已回复 | 计算回复率 |
| 仪表盘-拒绝 | 触达状态 = 拒绝 或 是否触发勿扰 = 是 | 勿扰检查 |
| 仪表盘-错误联系方式 | 触达状态 = 错误联系方式 | 反馈给线索补全 |

### 4.3 渠道来源表视图

| 视图名 | 筛选条件 | 用途 |
|---|---|---|
| 可用于 PoC | 是否进入 PoC = 是，风险等级 in Low/Medium | 当前可采渠道 |
| 政策研究 | 风险等级 = High | 不进入自动化任务 |
| 禁止使用 | 风险等级 = Forbidden | 阻断 |

## 5. 指标计算公式

以下为手工口径，可用飞书统计、透视表或导出 CSV 后计算。

| 指标 | 公式 |
|---|---|
| A 级占比 | A 级数 / 总候选线索数 |
| B 级占比 | B 级数 / 总候选线索数 |
| C 级占比 | C 级数 / 总候选线索数 |
| B/C 级占比 | (B 级数 + C 级数) / 总候选线索数 |
| 无效率 | Invalid 数 / 总候选线索数 |
| Watch 占比 | Watch 数 / 总候选线索数 |
| 回复率 | 已回复数 / 已发送数 |
| 拒绝率 | 拒绝数 / 已发送数 |
| 错误联系方式率 | 错误联系方式数 / 已发送数 |
| 每条候选线索成本 | 总成本 / 总候选线索数 |
| 每条有效线索成本 | 总成本 / (B 级数 + C 级数) |
| 每条回复成本 | 总成本 / 已回复数 |
| 每个销售机会成本 | 总成本 / C 级数 |

除数为 0 时显示 `N/A`，不要显示 0%，避免误导。

## 6. 渠道结论规则

| 结论 | 判定 |
|---|---|
| 继续 | Low/Medium 风险，B/C 级占比稳定，未出现风险事件 |
| 调整 | 有有效线索但无效率高，需调整关键词、地区或排除词 |
| 暂停 | 采集 30-50 条仍没有 B 级线索，或无效率过高 |
| 政策研究 | High 风险渠道，仅保留政策研究和人工小样本 |
| 禁止 | Forbidden 或出现明显违规风险 |

## 7. Go/No-Go 指标看板

| 决策项 | Go 标准 | 当前值 | 结论 |
|---|---|---:|---|
| 可持续低/中风险渠道 | ≥3 个 |  |  |
| 总线索数 | 100-200 |  |  |
| B 级比例 | 30%-50% |  |  |
| 人工触达样本 | 10-20 |  |  |
| 真实回复或明确负反馈 | 有 |  |  |
| 平台投诉/封禁 | 0 |  |  |
| 高风险自动任务 | 0 |  |  |
| 客服/销售承接意愿 | 愿意 |  |  |
| 贸易路径可行性 | 初步可行 |  |  |

## 8. 暂停条件看板

出现以下任一项，应进入暂停或转向讨论：

| 暂停条件 | 是否触发 | 证据 |
|---|---|---|
| B 级线索低于 20% |  |  |
| 触达几乎无回复或多数明确拒绝 |  |  |
| 出现投诉、封禁、访问异常或明显违规风险 |  |  |
| 销售团队不愿承接 |  |  |
| 贸易、支付、物流或清关路径不可行 |  |  |
| High/Forbidden 渠道成为主要线索来源 |  |  |

## 9. 截图汇报建议

PoC 汇报建议截图 4 个区域：

1. 顶部 KPI：总量、B 级比例、回复率、风险事件。
2. 渠道产出表：按渠道展示 B/C 级占比和无效率。
3. 触达漏斗：待触达、已发送、已回复、拒绝、错误联系方式。
4. Go/No-Go 看板：是否进入 MVP、重跑 PoC 或暂停。
