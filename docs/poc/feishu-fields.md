# PoC 飞书五张表字段字典与创建实施方案

创建日期：2026-05-26  
对应 Story：E0-S1 创建飞书五张表

## 1. 目标

本文件用于在飞书多维表格中创建海外车辆采购 AI 获客 PoC 的五张核心表：

1. 客户线索表。
2. 渠道来源表。
3. 车源/报价表。
4. 触达记录表。
5. 话术库表。

PoC 阶段先用飞书承载业务闭环，不开发 CRM、不自动发送消息、不做双向同步。MVP 阶段再将其中的关键字段单向同步到 PostgreSQL。

## 2. 通用创建规则

### 2.1 命名规范

飞书多维表格建议命名：

`俄罗斯车辆采购AI获客PoC`

五张表建议命名：

- `客户线索`
- `渠道来源`
- `车源报价`
- `触达记录`
- `话术库`

### 2.2 字段类型约定

| 类型 | 使用场景 |
|---|---|
| 文本 | 客户名称、备注、AI 建议、证据摘要 |
| 单选 | 等级、状态、风险等级、客户类型 |
| 多选 | 允许动作、禁止动作、适用车型、适用渠道 |
| URL | 来源链接、政策来源、图片/视频链接 |
| 电话 | 电话、WhatsApp、Telegram 号码等 |
| 邮箱 | 邮箱 |
| 人员 | 负责人、发送人、审核人 |
| 日期时间 | 采集时间、发送时间、有效期、更新时间 |
| 复选框 | 是否勿扰、是否可出口、是否已审核 |
| 关联记录 | 客户与来源、触达、车源、话术之间的关联 |
| 公式 | SLA、线索完整度、是否超时等后续可选字段 |

### 2.3 PostgreSQL 同步标记

字段字典中的“同步到 PostgreSQL”含义：

- `是`：MVP 数据库必须保留。
- `建议`：MVP 建议保留，后续做看板、AI 审计或复盘会用到。
- `否`：PoC 协作用字段，暂不进入数据库。

## 9. 飞书平台创建步骤

### 9.1 创建多维表格

1. 登录飞书。
2. 进入目标团队空间或项目文件夹。
3. 点击“新建”。
4. 选择“多维表格”。
5. 命名为：`俄罗斯车辆采购AI获客PoC`。
6. 将默认表重命名为 `客户线索`。

### 9.2 创建五张表

1. 在底部表格标签处点击“+”。
2. 依次新建并命名：
   - `渠道来源`
   - `车源报价`
   - `触达记录`
   - `话术库`
3. 确认五张表名称与本文件一致，便于后续同步映射。

### 9.3 配置字段

建议按以下顺序配置：

1. 先创建 `客户线索` 表的必填字段。
2. 再创建 `渠道来源` 表，并补充风险等级、允许动作、禁止动作。
3. 创建 `话术库` 表，先录入 3-5 条草稿话术。
4. 创建 `车源报价` 表，录入 3-5 条内部样例车源。
5. 创建 `触达记录` 表，并关联客户线索和话术库。

字段配置注意：

- 单选和多选字段必须提前配置可选项，避免后续录入不一致。
- `是否勿扰` 默认值为否。
- `线索状态` 默认值为新采集。
- `线索等级` 默认值可为 Watch 或 A，实际按复核结果调整。
- `人工确认` 默认值为否，外发前必须人工勾选。

### 9.4 配置推荐视图

在每张表左上角视图区域新建视图：

1. 选择“表格视图”。
2. 按本文件的推荐视图设置筛选条件。
3. 为不同角色设置常用视图：
   - 线索运营：待复核、待补全、无效。
   - 客服：B级线索、待触达、已回复。
   - 出口销售：C级线索、已回复、可参考车源。
   - 合规/风控：高风险来源、拒绝/勿扰、待合规审核。
   - 管理者：全部线索、渠道来源、已触达。

### 9.5 权限建议

PoC 阶段可先采用简单权限：

| 角色 | 权限建议 |
|---|---|
| 业务负责人 | 管理员 |
| 线索运营 | 可编辑客户线索、渠道来源 |
| AI/Agent 操作人员 | 可编辑客户线索、渠道来源、话术库草稿 |
| 客服 | 可查看客户线索，可编辑触达记录和勿扰字段 |
| 出口销售 | 可查看 C 级线索和车源报价，可编辑销售跟进备注 |
| 合规/风控 | 可编辑渠道风险、话术审核状态、勿扰检查 |

敏感控制：

- 不给外部人员开放表格权限。
- 不在表格中保存非公开、来源不明或不必要的敏感数据。
- 高风险渠道相关记录必须保留风险等级，不可混入 Low/Medium 视图。

### 9.6 样例数据验证

创建完成后，建议录入 5-10 条样例数据：

1. 2 条 A 级基础线索。
2. 2 条 B 级增强线索。
3. 1 条 C 级高意向线索。
4. 1 条 Invalid 无效线索。
5. 1 条 Watch 沉淀观察线索。
6. 1 条勿扰线索。
7. 1 条 High 风险渠道线索，用于验证高风险视图。

验证重点：

- 待复核视图是否排除勿扰线索。
- B级线索视图是否仅显示 B 级且非勿扰。
- C级线索视图是否仅显示 C 级且非勿扰。
- 勿扰视图是否能完整展示拒绝原因。
- 高风险来源视图是否能识别 High/Forbidden 渠道。
- 触达记录能否关联到客户线索。

## 10. 后续 PostgreSQL 映射建议

| 飞书表 | MVP 数据表建议 | 同步方式 |
|---|---|---|
| 客户线索 | `customers`, `lead_sources`, `contact_methods` | 单向同步 |
| 渠道来源 | `channel_risk_rules` | 单向同步 |
| 车源报价 | `inventory_items` | 单向同步 |
| 触达记录 | `outreach_records` | 单向同步 |
| 话术库 | `script_templates` 或 `knowledge_base_items` | 单向同步 |

同步原则：

- 飞书到 PostgreSQL 单向同步。
- MVP 不做双向同步。
- 勿扰状态同步后不得丢失。
- 来源证据和 AI 建议字段必须保留。
- 同步失败必须记录日志，不反写飞书。
