# Story E0-S4：建立 PoC 复盘模板

## 基本信息

- Epic：E0 PoC 运营底座
- Sprint：Sprint 2 PoC 触达与复盘
- 优先级：P0
- 状态：Done
- 负责人建议：业务负责人

## 用户故事

作为业务负责人，我希望 PoC 结束时有统一复盘模板，以便决定继续 MVP、重跑 PoC 或暂停。

## 业务价值

让阶段决策基于证据，而不是感觉。

## 依赖

- E0-S1 创建飞书五张表
- E4-S2 记录触达结果

## 任务清单

- [x] 设计 PoC 复盘模板。
- [x] 汇总渠道有效率、A/B/C 线索比例、触达回复、风险事件。
- [x] 收集客服和出口销售反馈。
- [x] 记录人工时间、AI/API 成本和工具成本。
- [x] 给出 Go、重跑 PoC、暂停俄罗斯方向三类结论。
- [x] 明确下一阶段调整建议。

## 验收标准

- 模板包含渠道有效率、A/B/C 线索比例、触达回复、风险事件、客服/销售反馈和 ROI。
- 模板包含三种结论：继续 MVP、调整后重跑 PoC、暂停俄罗斯方向。
- 模板包含暂停/转向条件。

## 非目标

- 不要求 PoC 复盘自动生成完整经营分析报告。

## QA / 风控检查

- [x] 必须记录是否出现账号限制、投诉或违规采集。
- [x] 未达 Go 条件不能直接进入 MVP。
