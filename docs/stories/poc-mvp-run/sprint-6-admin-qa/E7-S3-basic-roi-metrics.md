## 成本与 ROI 口径

- 人工成本：`labor_hours × hourly_rate`，通过 `POST /dashboard/roi-costs` 记录。
- AI/API 成本：直接记录 `amount`，类型为 `ai_api`。
- 工具成本：直接记录 `amount`，类型为 `tool`。
- 总成本：人工成本 + AI/API 成本 + 工具成本。
- 有效线索：B 级 + C 级客户。
- 回复数：触达记录状态为 `replied` 的记录。
- 销售机会：MVP 阶段按 C 级客户数统计。
- 每条有效线索成本：`total_cost / effective_lead_count`。
- 每条回复成本：`total_cost / reply_count`。
- 每个销售机会成本：`total_cost / sales_opportunity_count`。
- 渠道过滤：成本、有效线索和回复分母都按渠道过滤，避免用全局线索数稀释单渠道 ROI。
- 合规护栏：ROI 只用于经营判断，不能作为绕过 High/Forbidden、勿扰、C 级报价前合规复核等限制的理由。
