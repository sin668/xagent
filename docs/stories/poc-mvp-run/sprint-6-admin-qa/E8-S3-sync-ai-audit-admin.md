## 用户故事

作为系统运营，我希望查看飞书同步和 AI 执行日志。

## 业务价值

确保数据和 AI 判断可追踪、可排错。

## 验收标准

- 显示最近同步时间。
- 显示同步成功/失败条数。
- 显示 AI 执行任务、模型、状态和风险。
- 被阻断任务有清晰原因。

## 非目标

- 不做复杂日志搜索平台。

## QA / 风控检查

- [x] AI 审计日志不可被普通用户删除。
- [x] 被阻断任务必须保留原因。

## 实现记录

- 新增 `GET /sync/audit-dashboard` 只读聚合接口，展示飞书同步日志与 AI 审计日志。
- 同步摘要包含最近同步时间、同步成功条数、同步失败条数。
- 同步日志包含对象名、状态、成功/失败条数和失败原因。
- AI 审计日志包含任务类型、模型、prompt 版本、状态、风险状态、来源链接、阻断原因和执行时间。
- 支持按 `task_type` 与 `status` 过滤 AI 审计日志；接口额外支持 `source_name`、`model_name` 便于运维排查和测试隔离。
- 管理后台新增“飞书同步与 AI 审计”区块，对齐 `admin-sync.html` 原型的同步日志与 AI 执行审计结构。
- 未新增任何删除 AI 审计日志的 API 或前端动作。
