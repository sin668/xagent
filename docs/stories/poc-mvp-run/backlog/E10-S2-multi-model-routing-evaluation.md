# Story E10-S2：多模型路由和评测

## 基本信息

- Epic：E10 可扩展配置
- Sprint：Backlog
- 优先级：P2
- 状态：Draft
- 负责人建议：AI 负责人 + 后端工程

## 用户故事

作为 AI 负责人，我希望不同任务未来可使用不同模型，并有轻量评测，以便控制成本和质量。

## 业务价值

为后续规模化和模型优化预留空间。

## 进入条件

- AI 抽取、分类、翻译、话术任务量稳定。
- 已积累足够人工复核样本。
- 成本或质量问题成为明确瓶颈。

## 任务清单

- [ ] 按抽取、翻译、分类、话术定义任务类型。
- [ ] 支持不同任务选择不同模型。
- [ ] 建立小样本评测集。
- [ ] 评测抽取准确率、俄语质量、分类一致性。
- [ ] 展示模型成本和质量对比。

## 验收标准

- 可按抽取、翻译、分类、话术选择模型。
- 有小样本评测集。
- 评测维度包含抽取准确率、俄语质量、分类一致性。

## 非目标

- V1 不做复杂模型平台。

## QA / 风控检查

- [ ] 模型输出仍需保留审计。
- [ ] 模型切换不得降低合规护栏。
