# Story E10-S1：国家/地区配置

## 基本信息

- Epic：E10 可扩展配置
- Sprint：Backlog
- 优先级：P2
- 状态：Draft
- 负责人建议：产品负责人 + 后端工程

## 用户故事

作为产品负责人，我希望系统未来可配置国家、语言、时区、币种和渠道，以便从俄罗斯扩展到中亚市场。

## 业务价值

避免系统被俄罗斯市场写死。

## 进入条件

- 俄罗斯 MVP 跑通完整闭环。
- 已确认第二市场国家。
- 目标国家渠道和合规初判完成。

## 任务清单

- [ ] 设计国家配置模型。
- [ ] 支持国家、语言、时区、币种、主要城市。
- [ ] 支持国家级渠道列表和风险规则。
- [ ] 支持国家级话术语言。
- [ ] 将俄罗斯配置迁移为默认国家配置。

## 验收标准

- 可配置国家、语言、时区、币种、主要城市。
- 可配置国家级渠道列表和风险规则。
- 可配置国家级话术语言。

## 非目标

- V1 MVP 不做多国家完整上线。

## QA / 风控检查

- [ ] 新国家上线前必须完成渠道风险登记。
- [ ] 不允许复用俄罗斯合规判断替代新国家判断。
