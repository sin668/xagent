# Story E5-S2：线索与车源匹配

## 基本信息

- Epic：E5 车源/报价匹配
- Sprint：Sprint 5 MVP 业务闭环
- 优先级：P1
- 状态：Done
- 负责人建议：后端工程 + 出口销售

## 用户故事

作为销售，我希望看到客户需求对应的推荐车源，以便快速判断是否推进报价。

## 业务价值

提升 C 级线索转销售机会的效率。

## 依赖

- E5-S1 维护轻量车源/报价表
- E3-S3 移动端线索详情

## 任务清单

- [x] 定义轻量匹配规则：车型、年份、价格区间、可出口状态。
- [x] 在线索详情展示推荐车源。
- [x] 展示推荐理由。
- [x] 展示价格有效期和可出口状态。
- [x] 对需要合规复核的车源给出风险提示。
- [x] 支持销售选择“推进报价”或“暂不匹配”。

## 验收标准

- 线索详情可显示推荐车源。
- 推荐理由包含车型、年份、车况、价格有效期和可出口状态。
- 需要合规复核的车源有风险提示。

## 非目标

- 不做自动报价。
- 不做复杂推荐算法。

## QA / 风控检查

- [x] 推荐车源不得等同于正式报价。
- [x] C 级报价前必须走合规复核。

## 交付说明

- 后端新增 `lead_inventory_matches` 表，记录线索与车源推荐、推荐理由、风险提示和销售决策。
- 新增 `POST /inventory/matches/{customer_id}/recommendations`，按车型、年份、价格区间、可出口和 E5-S1 报价安全规则生成轻量推荐。
- 新增 `POST /inventory/matches/{match_id}/decision`，支持销售选择 `advance_quote` 或 `not_match`。
- 推进报价只进入“报价前合规复核”门禁，接口返回 `formal_quote_allowed=false`，不形成正式报价。
- 移动端线索详情展示推荐车源、推荐理由、价格有效期、可出口状态、合规风险提示和“推荐不等同正式报价”声明。
