# Story E4-S2：触达记录

## 基本信息

- Epic：E4 AI 触达助手
- Sprint：Sprint 5 MVP 业务闭环
- 优先级：P0
- 状态：Done
- 负责人建议：前端工程 + 后端工程 + 客服负责人

## 用户故事

作为客服，我希望在系统中创建和查看触达记录，以便追踪客户回复和下一步动作。

## 业务价值

让线索从“可看”进入“可跟进、可交接、可统计”的业务闭环。

## 依赖

- E4-S1 生成俄语触达草稿接入
- E6-S2 勿扰机制

## 任务清单

- [x] 设计触达记录创建接口。
- [x] 在移动端实现触达记录表单。
- [x] 支持已发送、已回复、拒绝、无回复、错误联系方式等状态。
- [x] 支持记录下一步动作和负责人。
- [x] 客户拒绝后联动标记勿扰。
- [x] 在客户详情页展示触达历史。
- [x] 触达数据进入管理后台指标。

## 验收标准

- B 级线索可由客服触达并记录结果。
- 支持回复状态：已发送、已回复、拒绝、无回复、错误联系方式。
- 客户拒绝后自动标记勿扰。
- 触达记录可在客户详情页查看。

## 非目标

- 不接入完整邮件系统作为 P0。
- 不做自动发送。

## QA / 风控检查

- [x] 勿扰客户无法新增触达记录。
- [x] 已发送必须对应人工确认动作。

## 交付说明

- 后端新增 `POST /customers/{customer_id}/outreach-records` 与 `GET /customers/{customer_id}/outreach-records`。
- PostgreSQL `outreach_records` 支持 `no_response`、`bad_contact` 状态，并新增 `owner` 字段用于负责人落库。
- 移动端触达助手支持人工登记触达状态、负责人、下一步动作、结果摘要。
- 移动端客户详情页展示触达历史。
- 系统不自动发送邮件或社交消息，仅记录人工确认后的触达结果。
