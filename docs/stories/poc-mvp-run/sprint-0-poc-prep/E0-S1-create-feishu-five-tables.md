## 用户故事

作为线索运营，我希望先用飞书表格承载客户线索、渠道、车源、触达和话术，以便在不开发 CRM 的情况下跑通 PoC。

## 业务价值

快速启动 PoC，验证字段、流程、人工协作和后续 MVP 数据模型。

- 无

- [x] 创建客户线索表。
- [x] 创建渠道来源表。
- [x] 创建车源/报价表。
- [x] 创建触达记录表。

- [x] 为渠道来源表配置风险等级、允许动作、禁止动作、政策来源字段。

## 验收标准

- 已创建客户线索表、渠道来源表、车源/报价表、触达记录表、话术库表。
- 客户线索表至少包含客户名称、国家、城市、客户类型、等级、状态、联系方式、来源链接、负责人、AI 建议、勿扰状态。
- 渠道来源表至少包含平台、关键词、风险等级、允许动作、禁止动作、政策来源。
- 触达记录表至少包含客户、渠道、话术版本、发送人、发送时间、回复状态、下一步动作。
- 业务负责人确认 PoC 可以基于这五张表执行。

## 非目标

- 不做飞书到 CRM 的双向同步。
- 不做自动发送消息。
- 不做复杂权限系统。
