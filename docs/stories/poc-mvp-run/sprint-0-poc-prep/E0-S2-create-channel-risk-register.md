- Epic：E0 PoC 运营底座
- Sprint：Sprint 0 PoC 准备
- 优先级：P0

- 负责人建议：合规/风控 + AI/Agent 操作负责人

## 用户故事

作为合规负责人，我希望每个渠道都有风险等级和允许/禁止动作，以便 Agent 和人工操作不越界。

## 业务价值

降低平台封禁、投诉、品牌风险和跨境业务合规风险。

## 依赖

- E0-S1 创建飞书五张表

## 任务清单

- [x] 建立渠道风险登记表字段。
- [x] 为官网、公开目录、搜索引擎、地图结果建立 Low/Medium 风险初判。
- [x] 为 YouTube、Telegram、X、Facebook/Instagram、Avito、VK、Auto.ru、Drom 建立风险初判。
- [x] 写明每个渠道允许动作。
- [x] 写明每个渠道禁止动作。
- [x] 标注政策来源链接、证据截图或人工备注。
- [x] 定义 High 渠道在 PoC 中仅可用于政策研究和人工小样本。

## 验收标准

- 至少覆盖官网、公开目录、搜索引擎、Google/Yandex 地图结果、YouTube、Telegram、X、Facebook/Instagram、Avito、VK、Auto.ru、Drom。
- 每个渠道标注 Low、Medium、High、Forbidden 之一。
- 每个渠道明确允许动作和禁止动作。
- High 风险渠道在 PoC 中不能进入自动化任务。

## 非目标

- 不承诺所有平台政策已完成最终法律审查。
- 不做自动平台政策监控。
