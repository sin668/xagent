# Story E0-S3：创建俄罗斯关键词库

## 基本信息

- Epic：E0 PoC 运营底座
- Sprint：Sprint 0 PoC 准备
- 优先级：P0
- 状态：Done
- 负责人建议：AI/Agent 操作负责人 + 线索运营

## 用户故事

作为 AI/Agent 操作负责人，我希望有俄语、英语、中文关键词库，以便稳定发现俄罗斯车商线索。

## 业务价值

减少临时搜索，提高线索发现的一致性、可复盘性和渠道扩展效率。

## 依赖

- E0-S1 创建飞书五张表

## 任务清单

- [x] 建立客户关键词：车商、经销商、二手车销售、进口车等。
- [x] 建立车辆关键词：二手车、准新车、库存车、品牌/车型等。
- [x] 建立渠道关键词：官网、地图、目录、社媒公开页等。
- [x] 建立排除关键词：维修、配件、保险、招聘、媒体等。
- [x] 建立地区关键词：Moscow、Saint Petersburg、Vladivostok、Novosibirsk、Kazan 等。
- [x] 为关键词配置语言、用途、适用渠道和有效率字段。

## 验收标准

- 关键词库包含客户关键词、车辆关键词、渠道关键词、排除关键词和地区关键词。
- 至少包含 Moscow、Saint Petersburg、Vladivostok、Novosibirsk、Kazan 等地区关键词。
- 每个关键词记录语言、用途、适用渠道和有效率字段。
- 关键词可直接用于 Sprint 1 公开渠道试采。

## 非目标

- 不要求第一版覆盖全部俄罗斯城市。
- 不追求关键词自动生成完整闭环。

## QA / 风控检查

- [x] 关键词不得诱导登录后批量采集。
- [x] 排除关键词必须覆盖明显非目标商家。
