# Story E2-S2：AI 线索分级建议

## 基本信息

- Epic：E2 AI 线索研究与清洗
- Sprint：Sprint 1 PoC 采集与 AI 清洗
- 优先级：P0
- 状态：Done
- 负责人建议：线索运营 + AI/Agent 操作负责人

## 用户故事

作为线索运营，我希望 AI 给出 A/B/C/Invalid/Watch 建议和原因，以便快速复核。

## 业务价值

提高线索分配效率，保护客服和出口销售注意力。

## 依赖

- E1-S1 定义客户主体模型
- E2-S1 AI 公开网页信息抽取

## 任务清单

- [x] 定义 A/B/C/Invalid/Watch 分级规则。
- [x] 定义 AI 输出：推荐等级、推荐原因、缺失信息、下一步动作、建议交付团队。
- [x] 将 B 级建议交付客服。
- [x] 将 C 级建议交付出口销售，但保留合规复核。
- [x] 将 Invalid 和 Watch 排除出触达队列。
- [x] 支持人工覆盖 AI 建议。
- [x] 记录人工覆盖原因。

## 验收标准

- AI 输出推荐等级、推荐原因、缺失信息、下一步动作和建议交付团队。
- A 级只入库补全。
- B 级建议交付客服。
- C 级建议交付出口销售。
- Invalid 和 Watch 不进入触达队列。
- 人工可覆盖 AI 建议，并记录覆盖原因。

## 非目标

- MVP 不做黑箱意向评分。
- AI 不直接决定最终交付。

## QA / 风控检查

- [x] C 级线索必须标记合规复核。
- [x] AI 评分必须可解释。
