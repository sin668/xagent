# Story E2-S1：AI 公开网页信息抽取

## 基本信息

- Epic：E2 AI 线索研究与清洗
- Sprint：Sprint 1 PoC 采集与 AI 清洗
- 优先级：P0
- 状态：Done
- 负责人建议：AI/Agent 操作负责人 + 线索运营

## 用户故事

作为线索运营，我希望 AI 从公开来源文本中抽取客户名称、城市、联系方式、经营信号和证据备注。

## 业务价值

减少手工复制、翻译和整理，提高 PoC 线索采集效率。

## 依赖

- E0-S2 创建 PoC 渠道风险登记表
- E0-S3 创建俄罗斯关键词库

## 任务清单

- [x] 选择 Low/Medium 风险公开来源做试采。
- [x] 定义 AI 抽取输入格式：来源链接、网页文本、渠道、关键词。
- [x] 定义 AI 输出字段：客户名称、国家、城市、类型、联系方式、活跃度、规模信号、进口/二手相关性、证据备注。
- [x] 要求缺失字段返回 Unknown 或空值。
- [x] 保存 AI 输入、输出、模型、时间、来源链接。
- [ ] 完成首批 50 条候选线索抽取。
- [ ] 人工抽样复核抽取准确性。

## 验收标准

- AI 只处理公开低/中风险来源。
- 输出客户名称、国家、城市、客户类型、联系方式、活跃度、规模信号、进口/二手相关性、证据备注。
- 缺失字段必须标记为 Unknown 或空值，不允许编造。
- 每次 AI 输出保存输入、输出、模型、时间、来源链接。

## 非目标

- 不自动登录平台。
- 不抓取非公开数据。
- 不做反爬规避。

## QA / 风控检查

- [x] 每条记录有来源链接。
- [x] AI 不得补写没有证据的联系方式。
- [x] High/Forbidden 渠道未进入批量任务。
