# Story E1-S3：实现基础去重规则

## 基本信息

- Epic：E1 线索数据模型与同步
- Sprint：Sprint 1 PoC 采集与 AI 清洗
- 优先级：P0
- 状态：Done
- 负责人建议：线索运营 + 后端工程预研

## 用户故事

作为线索运营，我希望系统提示疑似重复客户，以便人工合并或忽略。

## 业务价值

减少重复触达，提高 CRM 数据可信度。

## 依赖

- E1-S1 定义客户主体模型

## 任务清单

- [x] 定义强重复规则：客户名称 + 联系方式。
- [x] 定义疑似重复规则：客户名称 + 城市 + 来源域名。
- [x] 在飞书中先用公式或筛选辅助识别重复。
- [x] 设计 MVP 数据库中的重复候选字段。
- [x] 约定人工复核动作：合并、忽略、保留观察。
- [x] 合并时保留所有来源证据和联系方式。

## 验收标准

- 支持按客户名称 + 联系方式识别强重复。
- 支持按客户名称 + 城市 + 来源域名识别疑似重复。
- 疑似重复必须进入人工复核，不自动删除。
- 合并后保留所有来源证据和联系方式。

## 非目标

- MVP 不做复杂模糊匹配和机器学习去重。

## QA / 风控检查

- [x] 不自动删除线索。
- [x] 合并后来源证据不丢失。
