# 第二阶段小范围运行 LLM 来源发现 Agent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按 Story 逐个实现。每个 Story 完成后必须执行两轮独立评审，不得跳到下一个 Story。

**Goal:** 实现第二阶段小范围运行闭环：真实 LLM Provider 连接、Source Discovery Agent 自动新增来源候选、移动端来源审核、`LEAD_EXTRACTION` 自动消费来源池、任务调度与全链路审计。

**Architecture:** 后端以 FastAPI + SQLAlchemy + Alembic + PostgreSQL 为事实来源，Redis 提供短期锁，APScheduler 触发小范围定时任务。移动端以 uni-app(Vue3) 承载来源审核和 Agent 调用，管理后台以 Vue3/Vite 承载 LLM、Prompt、任务、成本和风险治理。

**Tech Stack:** Python, FastAPI, SQLAlchemy, Alembic, PostgreSQL, Redis, APScheduler, DeepSeek/OpenAI-compatible LLM client, uni-app Vue3, Vue3/Vite, Node v22.22.0.

- 代码实现前使用 `superpowers:test-driven-development`。
- 遇到 LLM、数据库、前后端联调异常时使用 `superpowers:systematic-debugging`。

- 每个 Story 完成后必须执行两轮独立多维度评审，并显式记录结论、发现项和修正结果。

- High/Forbidden 风险边界不得被自动化链路绕过。
