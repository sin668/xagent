# 海外车辆采购 AI 获客系统第五阶段 Go/No-Go 报告口径

创建日期：2026-06-06
适用阶段：第五阶段小范围运行
关联 Story：`docs/stories/phase-5-small-run/P5-E9-S3-phase5-go-no-go-report.md`

## 1. 目标

第五阶段 Go/No-Go 报告用于判断 Prompt 入库治理、邮件自动回复知识库、pgvector 语义召回、`EMAIL_REPLY` Agent、真实邮件发送和硬风险拦截是否达到受控扩大运行条件。

报告由 `apps/api` 统一生成，`apps/api` 是业务数据权威；`apps/admin`、移动端或 Agent 不得自行绕过本报告口径做自动发送放量判断。

## 2. API

接口：

```text
GET /dashboard/phase5-go-no-go-report
```

查询参数：

- `date_from`：统计开始日期，格式 `YYYY-MM-DD`。
- `date_to`：统计结束日期，格式 `YYYY-MM-DD`。
- `language`：可选，按回复语言过滤。
- `business_scene`：可选，按邮件回复业务场景过滤。
- `knowledge_collection_prefix`：可选，用于小范围运行或测试时限制知识库集合前缀。

## 3. 数据来源

报告必须回显数据来源，当前来源如下：

- `llm_prompt_templates`：Prompt 入库覆盖率。
- `knowledge_items`：已发布知识数量和可召回状态。
- `knowledge_embeddings`：embedding ready 率。
- `email_reply_drafts`：AI 生成成功率、人工采纳率、编辑幅度和硬拦截草稿。
- `email_send_attempts`：自动发送成功率、发送失败率和退信相关指标。
- `risk_events`：投诉、封禁、违规和建议暂停风险事件。

## 4. Go 条件

Go 条件与产品方案第 14 节一致：

- Prompt 入库覆盖率 `= 100%`。
- 已发布知识 embedding ready 率 `>= 95%`。
- AI 回复建议生成成功率 `>= 90%`。
- 人工采纳率 `>= 50%`。
- 自动发送硬拦截准确执行率 `= 100%`。
- DNC/勿扰客户不自动发送 `= 100%`。
- D/E 客户不自动发送 `= 100%`。
- 投诉、封禁、违规事件 `= 0`。
- 语言不确定、缺少同语言知识、知识召回不足时不自动发送 `= 100%`。

## 5. 结论枚举

### `go`

所有 Go 条件通过，可以进入受控扩大运行。

### `rerun_small_scope`

无硬风险事件，但存在质量或稳定性问题，需要重跑小范围：

- Prompt 覆盖不足。
- embedding ready 率不足或不稳定。
- AI 生成成功率不足。
- 人工采纳率低。
- 人工修改幅度高。
- 邮件发送失败率偏高但无风险事件。
- 知识库召回质量不足。

### `pause_auto_send`

发生以下任一情况，必须暂停自动发送：

- 投诉、封禁、违规或 `pause_suggested=true` 风险事件。
- DNC/勿扰误发。
- D/E 客户误发。
- 硬拦截命中后仍发送。
- 语言误判外发。
- 缺少知识证据仍自动发送。
- 自动回复承诺价格、库存、付款、合同或交付事项。

## 6. 口径说明

- Go/No-Go 报告中的回复质量指标只统计可回复草稿；硬拦截草稿不进入人工采纳率分母，避免将“正确阻断”误判为“人工未编辑采纳”。
- 硬拦截草稿单独进入 `hard_block_accuracy_rate`、`dnc_no_auto_send_rate`、`de_grade_no_auto_send_rate` 和 `knowledge_guardrail_no_auto_send_rate`。
- 日期窗口按 `created_at` 过滤，支持带时区与不带时区时间兼容。
- `recommended_actions` 必须给出中文行动建议，不允许只返回裸指标。
