# 海外车辆采购客户 AI 获客系统产品规划

创建日期：2026-05-26

## 1. 产品一句话

面向海外车辆出口业务的 AI 辅助 SDR/CRM 系统，先从俄罗斯二手车、准新车、库存车客户线索切入，帮助团队低风险发现、清洗、复核、触达和交付可跟进客户。

## 2. 产品定位

本产品不是自动群发工具，也不是通用爬虫系统。

它的第一阶段定位是：

- AI 线索研究员：从公开低风险渠道发现客户线索，提取结构化信息。
- AI 销售助理：生成客户摘要、推荐等级、缺失信息、俄语触达草稿和风险提示。
- 轻量 CRM：管理线索状态、分配、SLA、触达记录、客户交付和复盘指标。
- 合规护栏：限制高风险平台动作，保留 AI/Agent 执行证据，支持勿扰和人工合规复核。

## 3. 目标用户

| 用户 | 核心诉求 | 产品要解决的问题 |
|---|---|---|
| 业务负责人 | 判断俄罗斯方向是否值得继续投入 | 看渠道有效率、线索质量、回复情况、风险和 ROI |
| 线索运营 | 批量整理客户、去重、补全、初筛 | 减少手工整理，形成可交付 B/C 级线索 |
| AI/Agent 操作人员 | 运行搜索、抽取、分类、话术生成流程 | 有标准流程、提示词、日志和风险边界 |
| 客服 | 快速联系 B 级线索 | 看懂客户是谁、为什么值得联系、该怎么说 |
| 出口销售 | 承接高意向客户并推进报价 | 只接收 C 级高意向线索，看到需求、车源匹配和跟进历史 |
| 管理者 | 看团队执行和业务产出 | 看仪表盘、SLA、渠道 ROI 和销售机会 |
| 合规/风控 | 控制平台、数据、话术和贸易风险 | 有渠道风险等级、禁止动作、勿扰和合规复核节点 |

## 4. 用户旅程

### 4.1 线索运营旅程

1. 选择国家、渠道、关键词。
2. 从公开低风险渠道收集客户候选。
3. AI 抽取客户名称、联系方式、来源、经营信号。
4. AI 去重、补全、初筛和打标签。
5. 人工复核后确认 A/B/C 等级。
6. A 级入库补全，B 级交付客服，C 级交付出口销售。

### 4.2 客服旅程

1. 查看分配给自己的 B 级线索。
2. 查看客户基础信息、来源证据、AI 推荐理由。
3. 选择审核过的俄语话术。
4. 人工通过低风险渠道触达。
5. 记录回复、拒绝、下一步动作。
6. 客户有明确采购兴趣后升级为 C 级。

### 4.3 出口销售旅程

1. 查看 C 级高意向线索。
2. 查看客户需求、车源匹配、历史触达记录。
3. 判断车型、报价、物流、清关、付款可行性。
4. 报价或合同前触发合规复核。
5. 推进成交机会或退回观察。

## 5. 产品版本路线

### V0：PoC 运营版，2 周

目标：验证俄罗斯车商线索闭环是否成立。

产品形态：

- 飞书表格。
- 飞书文档 SOP。
- AI prompts。
- CSV/脚本辅助校验。
- 人工复核和人工触达。

核心产出：

- 俄罗斯渠道与竞品初版地图。
- 飞书五张表。
- 100-200 条初步线索。
- AI 清洗与补全流程。
- 10-20 条人工触达小样本。
- PoC 复盘报告。

### V1：MVP 轻量 CRM，4 周

目标：把 PoC 验证有效的流程产品化，支持团队试运行。

产品形态：

- Vue3 Web CRM。
- uni-app(Vue3) 移动端基础线索页。
- FastAPI + PostgreSQL。
- 飞书 API 单向同步。
- AI 建议面板和执行日志。
- 轻量仪表盘。

核心能力：

- 线索列表和状态看板。
- 线索详情页。
- AI 推荐、缺失信息、俄语话术、风险提示。
- 车源/报价匹配。
- 触达记录。
- B/C 级 SLA。
- 渠道风险配置。
- 勿扰/拒绝联系机制。

### V2：可复制市场版

目标：从俄罗斯扩展到中亚/其他区域。

新增能力：

- 国家/地区配置。
- 多语言话术配置。
- 渠道规则配置。
- 国家/区域权限。
- 更完善的关键词库和失败案例库。
- 多模型路由与轻量评测。

### V3：半自动运营版

目标：在合规和 ROI 验证后，提高运营效率。

新增能力：

- 低风险渠道人工确认后的半自动发送。
- 更细的意向规则。
- 销售机会预测。
- 多 Agent 分工。
- 官方 API 连接。
- 更完整的 CRM/ERP/库存系统集成。

## 7. 用户故事

### 线索运营

- 作为线索运营，我希望看到所有新采集线索，并按缺失信息排序，以便优先补全最可能成为 B 级的客户。
- 作为线索运营，我希望 AI 标出推荐等级和理由，以便快速决定是否交付客服。
- 作为线索运营，我希望系统提示重复客户，以便避免同一个车商被多次触达。

### 客服

- 作为客服，我希望只看到分配给我的 B 级线索，以便专注做初步沟通。
- 作为客服，我希望看到俄语触达草稿和风险提示，以便减少写话术时间并避免不当承诺。
- 作为客服，我希望客户拒绝后能一键标记勿扰，以便系统不再安排后续触达。

### 出口销售

- 作为出口销售，我希望只接收 C 级线索，以便把时间用于真实采购机会。
- 作为出口销售，我希望看到客户需求和推荐车源，以便快速判断是否能报价。
- 作为出口销售，我希望报价前能看到合规复核状态，以便避免贸易和支付风险。

### 管理者

- 作为管理者，我希望看到渠道产出和回复率，以便决定继续投入哪些渠道。
- 作为管理者，我希望看到 SLA 超时和团队处理量，以便判断团队承接是否卡住。
- 作为管理者，我希望看到 PoC 是否达到进入 MVP 标准，以便做阶段性决策。

## 8. 核心指标

### PoC 指标

| 指标 | 目标 |
|---|---:|
| 可持续低/中风险渠道 | ≥3 个 |
| 总线索数 | 100-200 条 |
| B 级线索比例 | 30%-50% |
| 人工触达样本 | 10-20 条 |
| 平台投诉/封禁 | 0 |
| 每条有效线索成本 | PoC 复盘计算 |
| 每条回复成本 | PoC 复盘计算 |

### MVP 指标

| 指标 | 目标 |
|---|---:|
| B 级线索首次跟进 | 48 小时内 |
| C 级线索首次跟进 | 24 小时内 |
| 勿扰客户再次触达 | 0 |
| AI 建议保留审计记录 | 100% |
| 高风险渠道自动执行 | 0 |
| C 级报价前合规复核 | 100% |

## 11. 产品风险

| 风险 | 影响 | 产品应对 |
|---|---|---|
| 平台封禁或投诉 | 中断渠道、损害品牌 | 渠道风险等级、禁止动作、只做人工低风险触达 |
| 线索质量低 | 客服/销售不愿承接 | A/B/C 等级、人工复核、失败案例库 |
| AI 判断不可信 | 团队不采用 | AI 证据日志、推荐原因、人工复核 |
| 客户拒绝后继续触达 | 投诉和合规风险 | 勿扰状态、全渠道排除 |
| 俄罗斯贸易/支付/物流不可行 | 销售无法成交 | C 级报价/合同前合规复核 |
| CRM 过早开发 | 浪费研发资源 | PoC 通过后才启动 MVP |

## 12. 产品决策原则

1. 先验证业务闭环，再产品化系统。
2. 先可解释 AI，再批量自动化。
3. 先低风险渠道，再扩展高价值渠道。
4. 先人工确认触达，再考虑半自动触达。
5. 先看有效线索和回复，再看成交。
6. 任何平台规则、数据保护、贸易合规风险都优先于增长速度。
