# 海外车辆采购 AI 获客系统 - 第一阶段小范围运行 Sprint 规划

创建日期：2026-05-29  
来源：`docs/product/2026-05-29-海外车辆采购AI获客系统-第一阶段小范围运行Epic-Story拆解.md`

## 1. Sprint 划分原则

- 先数据底座，再业务规则，再人工复核，再 Agent 自动化。
- 每个 Sprint 都必须保留合规边界：不自动私信、不自动加好友、不登录批采、不反爬规避。
- 所有 AI 输出必须保留来源证据和审计。
- High 公开页只作为发现入口，未二次复核不得进入 core 或触达队列。
- 勿扰客户不得再次进入触达队列。
- C 级报价/合同前必须合规复核。

## 3. Sprint 1：数据底座与渠道规则

目标：

- PostgreSQL 具备第一阶段小范围运行所需的 raw/staging/core/audit/knowledge 基线。
- 能配置渠道计划。
- 能在后端强制执行允许/禁止动作。
- High 风险公开发现任务被隔离。

验收：

- Alembic migration 可在真实 PostgreSQL 执行。
- collection_tasks、candidate_urls、page_snapshots、staging_leads、audit/risk 基础日志可用。
- channel_plans 可配置并校验风险等级。
- High 默认 public_discovery_only，queue_eligible=false。

## 4. Sprint 2：staging 复核闭环

目标：

- 运营可以查看 staging 复核队列。
- 可以查看线索证据、AI 推荐、准入闸门。
- 可以人工复核晋级 core。
- 可以发现重复并保留所有证据。
- 可以暂停风险渠道。

验收：

- 无来源链接、无证据、High 未二次复核、Invalid/Watch、勿扰不得进入待触达。
- 晋级 core 写入客户、联系方式、来源和 review_logs。
- 风险事件可以暂停渠道。

## 5. Sprint 3：Agent 任务流与知识库

目标：

- Agent 可按渠道计划生成候选 URL。
- 可读取公开页面摘要。
- LLM 可抽取、分级并写入 staging。
- 失败案例进入质量反馈。
- PostgreSQL + pgvector 知识库可检索 approved 条目，并接入 LLM prompt。

验收：

- Agent 不登录、不绕过、不自动触达。
- LLM 输出 schema 校验失败不得写入 staging。
- RAG 引用写入 ai_audit_logs。

## 6. Sprint 4：指标看板与治理

目标：

- 管理者可查看每日候选、staging、core 有效线索。
- 运营可查看渠道有效率、B/C 比例、联系方式完整率、重复率。
- 合规可查看风险事件和暂停渠道。
- 业务负责人可查看 LLM 成本与人工复核效率。

验收：

- High 只读结果不得计入可触达有效线索。
- 风险事件不可物理删除。
- 每条 core 有效线索 AI 成本可统计。
