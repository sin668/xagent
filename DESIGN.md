# 海外车辆采购 AI 获客系统设计上下文

## 视觉方向

冷静、专业、运营控制台感。界面服务线索运营、客服、销售和合规人员的日常处理，不做营销页表达。整体贴近 iOS/Android/微信小程序常见移动工作台：状态栏、圆角手机外框、底部 Tab、紧凑卡片、清晰标签、可扫读的数字和风险状态。

## 场景句

线索运营在白天办公室和通勤路上用手机快速审核来源、线索和邮件回复，屏幕需要明亮、信息密度适中、风险提示醒目且不疲劳。

## 色彩策略

使用克制的冷灰白底和少量状态色：

- 页面底：`oklch(96% 0.012 245)`，接近冷灰蓝。
- 卡片面：`oklch(99% 0.006 245)`，避免纯白。
- 主文本：`oklch(24% 0.025 250)`。
- 次级文本：`oklch(53% 0.035 250)`。
- 蓝色：任务、AI、可查看。
- 绿色：通过、低风险、有效。
- 琥珀：待确认、Medium、人工复核。
- 红色：阻断、High、Forbidden、DNC。

## 组件原则

- 页面宽度按 430px 设计，桌面 H5 居中显示为真实手机预览。
- 设备圆角不低于 34px，主要页面圆角与原型一致。
- 卡片圆角 18px 到 26px；列表卡片用细边框和轻投影，不使用嵌套卡片。
- 标签保持胶囊形，文字短且状态语义明确。
- 关键页面顶部可使用真实车辆、展厅或运营场景图片，但图片只服务业务上下文，不遮挡主要操作。
- 底部 Tab 固定 5 项，保留首页、线索/来源、客户、邮件/任务、合规/洞察等主路径。

## 字体与排版

优先使用 Apple 系统字体、SF Pro、PingFang SC 和微信小程序可用中文字体。标题使用高字重，正文限制在 12px 到 15px 的工作台密度，关键数字 24px 到 28px，避免超大营销字。

## 交互与状态

- 页面必须包含加载、错误和空态文案。
- 所有写入动作必须体现人工操作者、审核备注或确认语义。
- 微信小程序兼容优先，避免依赖 DOM-only API 完成核心功能。
- 前端可展示真实 API 请求失败，但不应回退成无标注的 seed 成功态。
